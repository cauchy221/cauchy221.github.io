1.4 Auto-encoder 自编码器#

1.4.1 自编码器结构与概念#

{% asset_img 1.4.1.png %}

自编码器包含两个部分，编码器Encoder和解码器Decoder，我们通常会使用神经网络作为编码器与解码器。编码器将输入压缩成潜在空间表征，解码器能够重构来自潜在空间表征的输入。

1.4.2 为什么要使用自编码器#

可能会有一个误区：输入数据与经过自编码器的输出数据”看起来差不多“，为什么还要使用自编码器？

其实自编码器进行了一个压缩再解压的过程。当我们用大量原始数据输入神经网络时，可能会消耗大量的时间，效果也不一定好，因此我们通过自编码器将原始数据压缩一下。

自编码器相当于提取出了输入中最具代表性最重要的信息，而其它信息被忽略了。

因此，本论文中的AE检测器就是一个自编码器，它将前面提取到的特征作为输入，然后经过一个自编码器，提取到输入特征中最关键的最重要的特征并输出。因此如果是对抗样本，输入的特征向量与普通样本的特征向量不一致（因为我们在生成AE的时候加了一段代码），但是经过自编码器输出的特征向量却是一样的（因为AE与普通样本中的关键特征都一样），所以我们可以比对输入特征向量与输出特征向量 的差异，找出谁是对抗样本。

1.5 Random walks 随机游走#

随机游走属于图嵌入方法的一种。图嵌入本身是表示学习，主要目的在于将图中的结点/图表示成低维、实值、稠密的向量形式，使得到的向量能够做进一步的推理，以更好的实现下游任务。

而随机游走的思想就是从一个顶点出发，按照一定的概率随机移动到一个邻接点，并将该结点作为新的当前结点，如此循环执行若干步，得到一条游走路径。然后把这个路径视为一个“句子”，用word2vec可以得到嵌入结果。

1.6 N-gram模型#

n-gram是nlp领域中涉及到的一个模型，可以用于统计一个句子出现的概率。

设一个句子S由w1, w2…..wn组成，那么我们可以应用条件概率，得到这个句子出现的可能性：

{% asset_img 1.6.png %}

但如果真的直接使用这个方程，对每个词要考虑它前面的所有词，在实际中意义不大且不好计算。我们可以基于马尔科夫假设来做化简。马尔科夫假设是指，每个词出现的概率只跟它前面的少数几个词有关。

常见的n-gram有1-gram, 2-gram和3-gram:

  • 对于1-gram，每个词都是独立分布的，也就是P(A,B,C) = P(A)P(B)P(C)

    比如语句：“猫，跳上，椅子” ，P(A=“猫”，B=“跳上”，C=“椅子”) = P(“猫”)P(“跳上”)P(“椅子”)。其中各个词的数量数语料库中统计的数量如下表：

猫跳上椅子
131623

　　可以求出P(A,B,C) = P(A)P(B)P(C) =13/M * 16/M * 23/M

  • 对于2-gram，每个词都与它左边的最近的一个词有关联，也就是P(A,B,C) = P(A)P(B|A)P(C|B)

    比如语句：“猫，跳上，椅子” ，P(A=“猫”，B=“跳上”，C=“椅子”) = P(“猫”)P(“跳上”|“猫”)P(“椅子”|“跳上”)。其中各个词的数量数语料库中统计的数量如下表：

    猫跳上椅子
    猫091
    跳上0315
    椅子000

    可以求出P(A,B,C) = P(A)P(B|A)P(C|B)，其中p(A) = 13/M，P(B|A) =9/13，p(C|B) = 15/16

  • 对于3-gram，每个词都与它左边的最近的两个词有关联。计算同上。

1.7 Centrality factor#

关于网络中心度的刻画，有三个常用的指标：

  • Degree centrality

用顶点的度表示Degree centrality。度高的结点(比如基站，无线网络中的接入点)更容易充当信息交换的角色。

{% asset_img 1.7-1.png %}

  • Closeness centrality

结点接近程度(Closeness centrality)，可以用结点间距离(geodesic distance，两个顶点之间最短路径中所包含边的数目)来表征。比如，一个结点到其他结点的最短路径都很短，那么该结点的Closeness就高。这个指标可以用来衡量信息从该结点传输到其他结点的时间长短。将一个结点到所有其他结点的最短距离累加起来取倒数，即为Closeness centrality。

{% asset_img 1.7-2.png %}

  • Betweenness centrality

一个结点如果经常出现在其他结点间最短距离路径中(即最短距离的路径经常包含该结点)，那么说明该结点更有能力促进其他结点间通信。计算公式如下，分母表示所有结点间最短路径的数目累加，分子表示所有结点间最短路径(该路径包含节点pi)的数目累加。

1.8 BFS 广度优先搜索#

之前数据结构课上已经讲解了BFS的流程，现在再做一个简单的介绍。

搜索顺序如下：假设从点P开始，点A, B, C与P相连，那么我们在找到P之后下一步可以找到A, B, C; 假设D, E与A相连，那么再下一步可以找到D, E，接下来可以去找与B直接相连的点、与C直接相连的点，以此类推去找后面几层的点，直到搜索完全部的结点。

2 总体介绍#

2.1 目前存在的问题和已有算法的缺陷#

目前在物联网界，针对malware detection和classification的算法对于对抗样本的干扰很脆弱，因此遇到对抗样本时，恶意软件的误分类率高；

为了解决前面提到的 ”对抗样本影响恶意软件分类“ 的问题，前期的一些工作已经利用不同的机器学习和深度学习方法在程序分析上进行恶意软件检测，其中主要包括动态分析和静态分析两类方法。

（1）Dynamic analysis

  • 需要执行恶意软件来提取特征，再将提取到的特征放入机器学习模型中；
  • 缺点：（1）复杂（2）消耗时间（3）可扩展性差（不能广泛适用）

（2）Static analysis

  • 不需要执行软件来提取特征，而是从软件的静态二进制文件中提取程序内容；
  • 其中有一种效果较好的方法：使用CFG控制流图（也是Soteria选择的方法）；
  • 但是目前采用CFG方法的算法，并不能检测出AEs；

有一些算法采用了机器学习的模型，但是由于机器学习模型的输出很大程度上取决于其输入，因此非常容易受到对抗样本的干扰。目前的算法都未能很好的解决这个问题。

2.2 Soteria框架#

针对上面提到的 ”对抗样本干扰“、”动态分析静态分析“和”机器学习缺陷“ 等相关问题，本论文提出了Soteria，致力于在基于CFG的恶意软件分类问题中，解决对抗样本造成的干扰问题。

{% asset_img 2.2.png %}

我们可以根据这张framwork了解Soteria具体的结构与工作流程。

Soteria一共包含三大的个部分：(1)特征提取; (2)对抗样本检测; (3)恶意软件分类；

下面我们对这三个部分进行一个简单介绍。

2.2.1 特征提取#

Soteria通过CFG控制流图来提取特征；具体包括首先分别用density-based 和 level-based两种方式来标记软件样本的CFG，再通过随机游走的方法，提取出一系列序列向量，最后使用n-gram模型来表示特征。

2.2.2 对抗样本检测#

由于我们后续的恶意软件分类器是由 “无对抗样本的数据集” 训练的，因此我们在分类之前要首先检测出对抗样本，并过滤这些对抗样本，只将合法（即不是对抗样本）软件样本送入后面的分类器，用于消除分类器针对对抗样本的脆弱性。

2.2.3 分类器#

Soteria使用多个CNN的组合，分别针对density-based 和 level-based。事实上，分类器是可以根据使用者不同的需求替换成其它方式的。

2.3 Soteria的效果#

（1）通过随机游走给使用到的分类特征提供了一个简单而有效的随机化特性，并通过首先检测出是否为对抗样本，有效避免了对抗样本对于后续分类的误导；

（2）用到了深度学习的方法，包含：

  • 一个自编码器auto-encoder，用来检测对抗样本；
  • 一个CNN架构，用来检测和分类恶意软件样本；

2.4 实验样本和结果#

  • 用到了16814个IoT样本，并与其它SOTA算法对比；
  • 对抗样本检测正确率97.79%；
  • 恶意软件分类正确率99.91%；

3 背景介绍#

由于我们的训练样本需要考虑加入一定的对抗样本，首先我们应明确现存的各种对抗样本生成方法，并选择一个合适的方法。

3.1 对抗样本的生成方法#

主要包括改变恶意软件的二进制层和改变其代码层这两种方法来生成。

3.1.1 Binary-level AEs#

对于基于二进制层的修改，也包含了以下两种方法： （1）在编译的时候改变一些字节，但是不考虑这些字节的功能和目的；

（2）在二进制文件的不可达部分 （对于给定操作来说，如果没有有效的控制流路径包含该操作，则其是不可达）增加一段良性的字节，从而改变后续提取出来的特征，造成误分类。但是这一种方法其实是impratical的（后面会讲）；

3.1.2 Code-level AEs#

对于代码层也有两种方法： （1）改变代码的结构；

比如说基于CFG的方法，就是通过增加或拆分函数导致代码结构改变，进而导致产生的样本的特征空间发生变化；

（2）插入一段外部代码以改变原始代码；

3.2 实际可行与不可行的AEs#

虽然根据上面的描述有许多方法能够生成对抗样本，但在实际操作中，为了使被通过某种方法改变为对抗样本的恶意软件依然是可执行（未被破坏）的，有一些论文中提到的生成方法（即上面提到的部分方法）是实际不可行的；

3.2.1 Impratical#

（1）二进制层：所有通过注入新的字节这种方法（如添加新的段落或者在文件尾部添加），在二进制文件层生成对抗样本的，都是实际不可行的。根据3.1.1的描述，我们没有选择Binary-level AEs；

（2）代码层：如果新增加的一段代码在编译时被认为是未使用的函数，这种也是实际不可行的；

3.2.2 Pratical#

如果生成AE时改变的部分是reachable的（即在代码流的一部分，也就是可以在CFG中直接或间接与entry相连的块），而且依然可执行（没有破坏原有代码），那么就是实际可行的；

3.2.3 总结#

事实上，都能通过部分基于二进制文件或者代码的方法来生产AEs，但是

  • 基于二进制文件层的，难以应用于 “产生精细的扰动”；

  • 近期研究表明，基于代码层，在原有代码后增加一段外部代码有以下三个优点：

    （1）造成的恶意软件分类的误分类率高；

    （2）不影响原有软件的功能；

    （3）影响原始样本的各种表现形式（包括提取出的特征）；

基于以上分析，Soteria采用了上述 ”注入外部代码“的方法来产生AEs，具体使用GEA会在后面详细介绍。

3.3 GEA介绍#

根据3.1与3.2的描述，Soteria选择使用”注入外部代码“的GEA (Graph Embedding and Augmentation)，下面对它进行详细描述：

3.3.1 优点#

（1）能生成可执行的对抗样本（即不会破坏原有的软件）；

（2）能够产生有目标的对抗攻击（即可以针对某一段代码or函数）；

（3）能够解决前面提到的 “某些文献提出的生成方法” 的局限性（即它是实际可行的）；

3.3.2 GEA与CFG结合#

{% asset_img 3.3.2.png %}

（1）将原始样本的代码和目标样本的代码结合，其中目标样本属于对抗者想要错误分类的类别；

（2）原始代码与注入代码的CFG共享一个entry block和一个exit block，这就导致了有两段分支，但只有左边的一段代码会被执行（也就是原始代码会被执行，因此不会影响软件的功能）；

（3）然而由于增加了一段外部代码，改变了整个CFG的结构，导致对后续提取出的特征造成影响（具有对抗者想要错误分类的类别的特征），进而导致了误分类；

3.4 Adversarial learning#

我们模型中的对抗样本检测器正式依靠这种 ”对抗性训练“来训练的，通过对同时含有干净样本和对抗样本的数据集进行训练来提高模型鲁棒性。

但是这样的对抗性学习存在着一定的限制性：

很多方法只使用了由一种AE生成方法得到的对抗样本来训练模型，导致对于其他方法生成的对抗样本检测的鲁棒性不高。特别是针对于基于代码而生成的对抗样本，攻击者完全可以稍微改变注入的代码，导致我们的检测器不能检测出来这个是对抗样本。

考虑到，对抗样本的良性和恶性部分可以同时存在于特征空间中，这使得我们需要找到一个方法：仅依赖于干净样本的结构就能检测出是否是对抗样本，这样，即便注入的代码变化（即附加在干净样本原始外的对抗样本代码变化），我们也不会受到干扰从而导致鲁棒性不强（因为干净样本原始代码部分没有变）。

本文的threat model采用上述提到的，基于代码、通过GEA改变CFG的方法来产生AEs的模型，而Soteria的目的就是在这个强大模型的存在下，提供一个精准且鲁棒的分类。

4 模型设计#

{% asset_img 4.png %}

对照这个framework，我们先大致介绍一下Soteria的模型框架：

4.1 Framework#

4.1.1 特征提取 Feature Extractor#

  • Soteria依赖于一个软件的CFG图来提取特征

  • 采用CFG的优点： （1）CFG通过展示与entry block直接或间接连接的代码，展示了可到达的块；

    （2）如果有采用注入一段代码的方法来生成的样本，并且新注入的代码在编译时被认为是未使用的函数，处于不可达部分，我们在提取样本时就会忽略这些代码段（因为CFG根本不会展示这些不可达块），因此消除noise injection和unused functions造成的影响；

4.1.2 对抗样本检测 Adversarial Example Detector#
  • 由于我们后续的恶意软件分类器是由 “无对抗样本的数据集” 训练的，因此我们在分类器之前先设置一个检测器，检测并过滤，只将合法样本送入后面的分类器，用于消除分类器对于对抗样本的脆弱性；
  • 不同于在3.4中提到的那些其它文献中使用同时含有干净样本和对抗样本的数据集进行训练的检测器，我们只使用 “无对抗样本的数据集”训练检测器（即仅依赖于干净样本的结构），但同时保持对抗样本和干净样本之间可区分的特征表示，以检测出潜在的对抗样本；
4.1.3 恶意软件分类器 Classifier#
  • 我们将过滤掉对抗样本后的样本输入由多个CNN的集合构成分类器中，分类器将这些样本分为良性或恶性（恶性又包含Gafgyt, Mirai, Tsunami三类）；
  • 虽然Soteria目前使用CNN作为分类器，它也可以用其他所需的方式替代；

4.2 这种架构的优点#

（1）在检测对抗样本之前我们提取出了样本的features。我们可以通过re-use这些features（将它们用于both对抗样本检测和恶意软件分类），而不用在样本检测后重新提取特征再放入分类器。这中顺序降低了特征提取的成本；

（2）用户可以自行选择使用哪些特征进行分类、分类器的参数甚至分类器的种类等；

5 特征提取#

{% asset_img 5.png %}

下面我们详细介绍Soteria的第一步：特征提取

5.1 样本预处理#

样本预处理主要是对软件样本的CFG进行标记，Soteria同时采用了 Density-based 和 Level-based的标记方法。

由之前的内容可知，当样本通过GEA转变为对抗样本时，我们的标记会发生改变（与原始样本不一致），从而改变了提取出来的特征。

5.1.1 Density-based Labeling（DBL）#
  • 基本概念#

    （1）Density of a node: 结点的度/边总数E;

    （2）Centrality factor (网络中心度) of a node: CF_vi = B_vi + C_vi;

    （3）Betweenness centrality B_vi: (对于所有的i != j != t, 从vj到vt之间, 经过vi的最短路径条数) / (vj与vt之间所有的最短路径条数);

    （4）Closeness centrality C_vi: 结点vi和其他所有结点之间的平均最短路径;

  • 操作步骤#

    （1）初步根据density对各个结点进行标号（most dense标0，其余的依次加1，最小的标|V|-1）；

    （2）若两个结点的density一样，则根据它们的CF_vi进行标记（CF_vi大的,标为小序号）；

    （3）若两个结点的CF_vi依然一样，再根据它们的level进行标号（将entry看作level1，level高的标为小序号）；

    （4）若两个结点的level依然一致，则将他们依次升序标号（因为此时两个结点相当于处于对称位置上，谁小谁大并无影响）；

    {% asset_img 5.1.1.png %}

如图所示，结点0和1都是most dense的结点（因为它们都与四块相连），但是计算得知结点0的网络中心度较高，因此它被标为0号。最后我们将10号分配给entry block，因为它是density最小且网络中心度最低的结点。

5.1.2 Level-based Labeling（LBL）#
  • 基本概念#

    （1）S_vi: 等于从entry node走到vi的最短步数;

    （2）Level of vi: 1 + S_vi (相当于把entry node看成level1，但是entry node被标记为0);

  • 操作步骤#

    （1）从入口块开始，通过广度优先搜索BFS的顺序对其它层次进行标记；

    （2）对于处于相同level的结点，我们再采用DBL中步骤2-4的方法进行标记；

{% asset_img 5.1.2.png %}

5.2 特征提取与表示#

在对样本的CFG进行标记后，我们采用随机游走和n-gram模型的方法提取并表示特征。

5.2.1 Random walk随机游走#

随机游走可以用于近似图的状态空间。

  • 操作流程#

    （1）假设有一个无向图G，我们首先将标记 mark 放在一个结点 vi 处，由 entry block 开始。每一步，这个标记 mark 都会以概率 p = 1/deg(vi) 随机走到一个邻接点 vj 。我们将每一步走到的点上的标记 label 记下来，得到一个序列 W，例如得到 W = “10 9 2 1 2 ……"。我们将随机游走的长度记为|W|，可见长度为|W|的随机游走记录下来的结点个数为|W|+1。在Soteria中，我们规定|W| = 5 x |V|；

    （2）每次游走都能得到一个长度为|W|、由各个结点的标记构成的序列向量。我们在DBL与LBL上分别游走10次，共得到20个序列向量。

随机游走的过程有助于将特征提取过程随机化。

5.2.2 n-gram模型#

n-gram模型可以被运用于文本、文档、图片等的特征表示。在node2vec的启发下，我们想到用n-gram来表示图。将n-gram用于之前通过随机游走得到的序列。

  • n-gram的原理#

    假设有一个字符串序列s，则它的n-gram就是将其拆分成多个长度为n的子串；

    例如：validation的Bi-gram：va, al, li, id……

  • Soteria的n-gram#

    在之前的随机游走过程中，我们得到了50+50=100个序列向量。对这些序列，我们分别提取他们n = 2, 3, 4时的n-gram，以得到很多短的子串（显然长度分别为2、3、4）；

    最后，我们基于W的频率，选择并使用每个LBL和DBL中的前500个判别特征（总共有500+500=1000个特征）

6 搭建对抗样本检测器模型#

{% asset_img 6.png %}

检测模型的核心是一个自编码器，其中包括一个输入层，一个输出层，三个隐藏层以及一个validation unit。神经网络中的五层是全连通的。

根据1.4.2中的分析，Soteria中的AE检测器就是一个自编码器，它将前面提取到的特征作为输入，然后经过一个自编码器，提取到输入特征中最关键的最重要的特征并输出。因此如果是对抗样本，输入的特征向量与输出的特征向量不一样（因为对抗样本在干净样本的基础上增加了新的代码用于误导分类器，导致其最关键特征是想要被分类到的那个类别的特征），所以我们可以比对输入与输出的差异，检测出谁是对抗样本。

  • Input layer

从n-gram模型中得到的500+500个特征向量作为输入；输入层由大小为1x1000的一维向量组成；

  • Hidden layer

隐藏层用于提取输入特征的最关键特征。先分别将输入的大小为1x1000的特征向量解码为1x2000和1x3000，然后再编码为1x2000。它消除了这些特征之间的依赖性；

  • Output layer

输入层与隐藏层的第三层完全连接。输出的向量的大小同样为1x1000（与输入的大小一致），然后将再被分为两个1x500的向量（分别对应DBL和LBL）；

  • Validation unit

通过输入与输出的均方根误差（RMSE）来计算reconstruction error（RE）。如果RMSE > 50%（这个是我们设置的阈值），则该样本被认为是对抗样本。

7 恶意软件分类器#

该分类器针对的是“非对抗样本”的干净样本，它将软件样本分为benign和malicious，其中malicious可分为Gafgyt，Mirai和Tsunami三类。

该分类器一共包含两个CNN分类器，且分别针对基于density和基于level的特征。

{% asset_img 7.png %}

7.1 CNN分类器#

{% asset_img 7.1.png %}

每一个CNN分类器的输入是一个大小为1x500的一维向量，即针对基于density的或基于level的特征；

分类器包含三块：卷积模块 (ConvB)1和2，以及一个分类模块 (CB)；

每一层分别使用ReLU作为激活函数，并以概率p随机dropout一些神经元的激活，使得模型的鲁棒性更强，同时避免过拟合；

同时采用池化的方法，步长（池化窗口移动大小）记为s，max-pooling size（池化窗口的大小）记为m；

7.1.1 ConvB1#

ConvB1的输入是前面提取到的特征，并且由两个连续的卷积层组成。每个卷积层都有深度为46，大小为1x3的过滤器，在s=1且没有填充的情况下进行卷积运算。经过两层卷积层后，得到46个大小为1x498的feature maps，并紧接着放入池化层和dropout防止过拟合（其中s=m=2，p=0.25），最后得到46个大小为1x249的feature maps。

7.1.2 ConvB2#

ConvB2除了含有深度为92，大小为1x3的过滤器之外，其余操作与ConvB1一致，最终得到92个大小为1x123的feature maps。

7.1.3 CB#

最后的分类模块的输入是将通过第二个卷积模块得到的feature maps全部一维化之后的特征，并将其送入大小为512，p=0.5的全连接层，最后输出送到一个softmax层进行分类（softmax相当于一个归一化函数，能够把样本在各个类别上的得分归一化为(0, 1)之间的概率，并通过概率解决多分类问题）。

7.2 Majority voting#

在前面的随机游走中提到，我们在DBL与LBL上分别游走10次，共得到20个序列向量。这些特征向量在经过后续的处理后被分别放入相应的CNN分类器，可以得到两个结果。而最终的结果基于majority voting机制，得票最高的类别作为该样本最后的分类。

11 一些讨论#

11.1 System Robustness#

  • 前面的评估表明，Soteria是robust的，并且能够以97.79%的准确率检测出对抗样本；
  • 在相同的训练和测试数据集上，Soteria的表现优于其他系统；
  • Soteria使用的majority voting机制使得对于Tsunami的分类率达到了100%的准确性，而其它已有模型因为使用数据集较小，对Tsunami的分类率较低；

11.2 Operation Mode for Detector#

在训练过程中，检测器不应该知道对抗样本和它们的生成模式，因为这将导致检测器的性能偏向于特定的攻击，降低对其它攻击的鲁棒性。

11.3 Adversarial Capabilities#

11.3.1 对抗样本能做到的#
  • 我们假设对抗样本可以访问原样本的源代码，并可以修改或者合并这些代码；
  • 而且对抗样本对Soteria的设计和内部结构有预先的了解；
11.3.2 对抗样本不能做到的#
  • 由于样本特征都是随机获得的，对抗样本不知道分类器会用到哪些特征；

11.4 Soteria的缺陷#

11.4.1 CFG-based Features#

CFG不一定反映了软件的实际代码。在不改变软件功能的情况下修改代码不会改变CFG的结构，因此对抗样本可能利用这一点来躲避对抗样本检测。

例如，向一个样本中注入一段不会产生新的分支、但会影响CFG结构的代码。虽然这种操作产生的对抗样本不会影响分类结果，但它会影响特征空间，导致我们需要重新训练Soteria。

11.4.2 Binary Obfuscation#

可以使用字符串混淆来隐藏部分代码，或者使用函数混淆，造成CFG的不完整，从而导致表示的特征的不完整，进而影响最终的分类结果。

因此，去除混淆是一个可以添加到模型中并不断完善的领域。
