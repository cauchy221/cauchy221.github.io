MANDA: On Adversarial Example Detection for Network Intrusion Detection System

1 一些后面用到的知识点#

1.1 Network Intrusion Detection System (NIDS)网络入侵检测系统#

1.2 Manifold Learning流形学习#

流形学习（manifold learning）是一类借鉴了拓扑流行概念的降维方法，在降维时，若低维流行嵌入到高维空间中，则数据样本在高维空间的分布虽然看上去十分复杂，但在局部上仍具有欧式空间（对现实空间的规则抽象和推广）的性质。

我们先来理解流形： 流形(manifold)是一般几何对象的总称，包括各种维度的曲线与曲面等，和一般的降维分析一样，流形学习是把一组在高维空间中的数据在低维空间中重新表示。不同之处是，在流形学习中假设：所处理的数据采样与一个潜在的流形上，或者说对于这组数据存在一个潜在的流形。

为什么要用流行学习降维：我所能观察到的数据其实是由一个低维流形映射到高维空间上的（一个立方体可以展开为平面）。由于数据内部特征的限制，一些高维中的数据会产生维度上的冗余，实际上只需要比较低的维度就能唯一的表示。 降维的举例：一个二维的圆，在整个二维空间中除了这个圆上的点之外的点，都是浪费的，用二维空间来表示这个圆是有冗余的；但如果用极坐标来表示这个圆，就只有半径这一个参数来确定圆。

通俗的说，流行学习可以概括为：在保持流形上点的某些几何性质特征（圆形、瑞士卷形）的情况下，找出一组对应的内蕴坐标，将流形尽量好的展开在低维平面上。

2 总体介绍#

2.1 目前已有的IDS (Intrusion Detection System)#

目前已存在的IDS主要分为两类：

（1）signature-based detection

  • 通过提取 traffic signature并将它们与之前建立好的一个 knowledge base对比进行检测
  • 缺点：只能检测已知的攻击

（2）anomaly-based detection

  • 试图通过测量和已建立的标准 traffic model之间的偏差来检测
  • 这个标准模型代表着正常的良性的 traffic，并用于以此评估某个 traffic的可信度
  • 目前这个标准模型通常通过ML方法（如LSTM, meta-learning, active learning）来训练

2.2 关于MANDA#

2.2.1 目前抵抗AEs的方法#

主要有两个：

（1）增强IDS模型对于对抗干扰的鲁棒性 （2）在样本进入IDS之前，先增设一个AE检测器

但是第一种方法往往针对不同的AE，需要定制不同的IDS，导致模型通用性差。因此本文选择第二章方法。

2.2.2 MANDA#

{% asset_img 2.2.2.png %}

由图可知，对抗样本（黑色圆形）往往与决策边界的距离很近。MANDA利用了由此带来的两点性质。

（1）矛盾性：虽然IDS模型会将对抗样本错误分类为良性，但是 manifold detector会将其分类到恶性样本中（因为AE在恶性样本的聚类中） （2）易变性：如果加一个小的噪音作为干扰，相比于干净样本，IDS对AE的分类结果更容易变化

2.2.3 三点贡献#
  • 第一个提出 problem space中AE对IDS的攻击，而不是特征空间；同样也是第一个提出对抗此类攻击的方法的
  • MANDA通过结合MANifold和DB来构建AE检测器
  • 在IDS和图片分类上的表现都很好

3 背景和相关工作#

3.1 Network Intrusion Attack#

3.1.1 以往的入侵攻击方式#

往往有三类常见的入侵攻击方式：

（1）扫描连接 比如扫描网络的端口，来找寻网络的脆弱点并发起攻击

（2）Advanced Persistent Threat (APT) 不依赖于运行代码，而是依赖于人类执行的相应操作

（3）DDoS 通过消耗网络资源来扰乱网络执行过程

3.1.2 最新的入侵攻击方式#

在机器学习发展之后，有很多AE攻击专门针对基于ML的IDS（具体方法见论文Ⅱ-A）

3.2 对抗样本#

关于对抗样本的研究主要有两个方向：（1）对抗样本攻击（2）对抗样本检测

3.2.1 AE attacks#

常见的AE攻击有以下四种：（1）FGSM（2）BIM（3）JSMA（4）CW

3.2.2 AE detection#

有很多对抗样本检测方法是仅仅针对于images的，因此在IDS上并不适用。但是基于statistical testing的方法并没有适用范围的限制，因此这写检测方法可以用于IDS（论文Ⅱ-B-AE detection列举了四种可行的方法）。

本文提出的MANDA是第一个针对于ML-based IDS的对抗样本检测器。

4 System Model and Threat Model#

4.1 Notation#

本论文构建的模型一共包含两个部分：

（1）MANDA

MANDA作为一个对抗样本检测器，放在整个模型的最前面，用于检测出对抗样本，并将这些AE剔除掉

  • clean input, clean sample
  • adversarial input, adversarial sample

（2）IDS Model

IDS Model作为一个入侵攻击检测器，用于判断输入的网络事件是良性的或者是恶性的。由于IDS Model放在MANDA的后面，它的输入不包含对抗样本。

  • 输入为一系列网络包（称为网络事件）
  • malicious input指由DDoS攻击或僵尸网络生成的恶意攻击，也被称为malicious data, malicious traffic, intrusion
  • benign traffic指由正常网络应用生成的时间，也被称为benign data, benign input, normal traffic

4.3 Threat Model#

4.3.1 站在攻击者的角度#

站在攻击者的角度，有三种攻击模式：

（1）white-box：攻击者知道IDS Model的架构和权值

（2）gray-box：攻击者知道架构，但不知道权值

（3）black-box：攻击者不知道架构也不知道权值

本篇论文考虑使用white-box攻击，因为这种方法产生的攻击最有力。但同时我们假设攻击者不知道模型的对抗样本检测方法。

4.3.3 选择的攻击（AE生成）方法#

本论文一共选择了四种常见的AE攻击方法

通常情况下，CW Attack的效果最好

（1）FGSM：fast gradient sign method

  • 详细介绍：https://blog.csdn.net/u014380165/article/details/90723948
  • 计算y关于x的梯度，并使x向着使J(θ, x, y)最大的方向移动

{% asset_img 4.3.3-1.png %}

  • 其中sign()函数计算梯度的方向。sign()函数是用来求数值符号的函数，比如对于大于0的输入，输出为1， 对于小于0的输入，输出为-1，对于等于0的输入，输出为0。之所以采用梯度方向而不是采用梯度值是为了控制扰动的L∞距离，这是FGSM算法的评价指标。
  • e为权重参数，可以用来控制攻击噪声的幅值，参数值越大，攻击强度也越大，肉眼也更容易观察到攻击噪声

（2）BIM：basic iterative method，是FGSM的多次迭代版本

  • 详细介绍：https://blog.csdn.net/u014380165/article/details/90724000
  • 可见就是FGSM的多次迭代

{% asset_img 4.3.3-2.png %}

  • a就是上面的权重e
  • Clip表示将溢出的数值用边界值代替。这是因为在迭代更新中，随着迭代次数的增加，部分像素值可能会溢出（比如超出0到1的范围），这时候就需要将这些值用0或1代替，这样最后才能生成有效的图像。

（3）JSMA：jacobian-based saliency map attack

  • 迭代地来找在每一步中对误分类贡献最大的特征
  • 对抗样本的显著图（saliency map）如下：

{% asset_img 4.3.3-3.png %}

  • 其中fj(x)表示将输入x分类到class-j的置信度。为了将原本属于class-j的x误分类为class-t，加入的扰动需要使fj(x)减小，ft(x)增大

（4）CW attack：Carlini and Wagner提出的

{% asset_img 4.3.3-4.png %}

  • x'-x对应着干净样本和对抗样本的差，差距越小越难检测
  • max{fi(x'):i != t}-ft(x')越小，表示将x误分类到class-t的可能性和分类到正确类别class-i的可能性越接近
  • k为置信度，k越大，那么模型分错的概率越大

5 The MANDA System#

由于我们的IDS系统的输入是problem-space中的流量，因此我们需要把已知的基于feature-space的AE生成算法转化成基于problem-space的算法。

5.1 Problem-Space AE Attack for IDS#

5.1.1 Problem-space and Feature-space#
  • IDS的problem-space指所有可能的网络流量实例，通常以一系列网络数据包的形式出现
  • IDS的feature-space指所有可能的特征向量，其中向量的数值可以表示数据包的长度、数据包到达的时间间隔等
5.1.2 在problem-space中生成AE的步骤#

（1）首先，通过一个输入的干净样本x，我们将其转化为基于feature-space的对抗样本x'

（2）然后，我们利用一个mapping函数，将x’映射到problem-space中，得到对应的对抗样本z'。干净样本x在problem-space中对应z

5.1.3 mapping函数#
  • 总体思想：通过消除不可微特征中的扰动，使mapping变成可微的
  • 具体操作如下：

（1）基于feature-space的对抗样本x’由两部分组成：{x'_diff, x'_non-diff}，另x'_non-diiff = x_non-diff

（2）由此直接可得，z'_non-diiff = z_non-diff

（3）我们将 f(x_diff) 的梯度从x反向传播到z，使z在problem-space中的变化遵循负梯度

  • IDS中的不可微特征包括协议类型，服务类型等等
  • 可见，消除这些不可微特征依然保留了干净样本x中的本质特性，使得AE能够成功误导分类器误判

5.2 Properties of AEs#

5.2.1 Manifold Learning流形学习#

流形学习可以概括为：在保持流形上点的某些几何性质特征（圆形、瑞士卷形）的情况下，找出一组对应的内蕴坐标，将流形尽量好的展开在低维平面上（具体见1.4）

在这里流形可以理解为簇。

设M为一个流形模型，则流行评估M(x)的输出为一个向量，该向量的各个分量表示位于各个子流形（即各个class）的概率

5.2.2 AE的性质#

不妨假设原来的x是恶性的，即c(x) = 1，需要将由x生成的对抗样本x’误分类到良性，即c(x') = 0

有以下两种情况：

（1）x’接近恶性类的流形，但是远离良性类的流形

（2）x’与恶性和良性的流形都接近

{% asset_img 5.2.2.png %}

可见以上两种情况是成功生成AE的，因为生成的对抗样本很好的保留了x中的本质特性。

相反，如果x’远离恶性的流形但是接近良性的流形，或者远离这两种流形，都是不成功的，因为没有很好的保留原有的恶性特质。

5.3 MANDA对抗样本检测器#

MANDA结合了两个独立的对抗样本检测方法，分别是Manifold和DB (desicion-boundary)

（1）Manifold

  • 综合IDS对x的分类结果和x的流形评估结果M(x)进行检测
  • 如果IDS对x的分类与M(x)不一致，则x很有可能是AE

（2）DB

  • 通过判断x是否在决策边界旁边来检测
  • 如果给x一个很小的Gaussian noise，c(x)发生了变化，说明x很可能在DB旁边，即x很有可能是AE
5.3.1 Manifold#
  • **针对Fig.3中的Case A ** 可以看到，在这个例子中，x’接近于恶性的流形，同时IDS分类器会将x分在良性类，产生了矛盾，因此检测出对抗样本
  • 如何得到良性类和恶性类的流形： 本论文利用了之前提出来的一个传感器学习模型，通过学习一组包含标记点和未标记的点的数据得到。该模型保证了局部一致性和全局一致性，即（1）相邻的点有同样的标签（2）拥有相同结构的点有同样的标签
  • 具体检测方法： 我们首先根据Algorithm 1计算score1，再将score1与设定的阈值进行比较（Algorithm 2）来判断是否为对抗样本

{% asset_img 5.3.1-1.png %}{% asset_img 5.3.1-2.png %}

5.3.2 DB#
  • 针对Fig.3中的Case B 可以看到，这个例子中良性和恶性的流形有重合部分
  • 我们做出以下假设来判断x‘是否为对抗样本：如果满足以下情况，则x’非常接近于决策边界

{% asset_img 5.3.2.png %}

  • 在接近DB的情况下，如果给对抗样本一个极小的扰动，其分类结果有很大可能性发生变化，此时的IDS分类器是不稳定的。我们通过测量这种不稳定性来检测对抗样本。
  • 由Algorithm 1可知，我们给x一个扰动N(0, σ^2)，并以此计算score2，并将score2与预先设置的阈值进行比较，判断是否为对抗样本
5.3.3 MANDA (Manifold + DB)#

（1）MANDA首先对训练集中的每一个样本x，计算他们的score1和score2，并记x的标签为label(x)

（2）再将[x, label(x)]转化为[score1, score2, label(x)]，以得到一组新的训练集

（3）最后，MANDA用这组新的训练集训练一个logistic回归模型，并最终用于对抗样本检测

6 实验结果#

6.1 Datasets#

6.1.1 NSL-KDD#
  • 互联网流量数据集，用于validation
  • 每一个样本都有4组字段：Intrinsic Characteristics, Content Characteristics, Time-based Characteristics, Host-based Characteristics
  • 该数据集一共包含四种攻击方式：DoS, Probing, Remote-to-Local (R2L), User-to-Root (U2R)，其中每一种下面有很多子类
  • 训练集有125973个样本，包含共24种攻击模式；测试集有22544个样本，包含共38种攻击模式
6.1.2 MINST#
  • 手写数字0-9，每个数字类有6000训练样本和1000测试样本
  • 每张图片都是28x28，且为灰色

6.2 Experiment Setting#

  • IDS Model针对NSL-KDD数据集采用了MLP多层感知机，包括1个输入层，1个隐藏层（50个神经元）和1个输出层
  • IDS Model针对MINST数据集采用CNN模型，包括4个卷积层（ReLU作激活函数），2个全连接层
  • 我们选取之前提过的FGSM, BIM, CW, JSMA作为AE生成方法，但是只将前三种放到problem-space里。在每组实验中，我们选择测试集中被IDS正确分类的样本来生成AE，并将这些对抗样本和随机选择的数量相同的干净样本混合，作为一个新的dataset来检测
  • 我们将MANDA与Artifact进行比较

6.3 Evaluation Metrics#

  • 选用TP, FP, TN, FN, TPR = TP/(TP+FN), FRP = FP/(TN+FP)
  • 绘制ROC曲线，计算AUC值 https://blog.csdn.net/qrlhl/article/details/50209841
